## 优先添加

### 1. SECURITY.md

涉及 LLM API 调用和 API key 处理，需要安全漏洞报告渠道，防止敏感信息通过公开 issue 泄露。

### 2. Issue templates

减少低质量 issue（如缺少版本号、日志信息）。建议两个模板:
- bug_report.yml (Bug Report)
- feature_request.yml (Feature Request)

### 3. CONTRIBUTING.md

明确贡献流程: fork → branch → PR → CI pass。项目已接受外部贡献（Awesome-AI-Meets-Biology），需要书面化流程。

## 可选

### 4. FUNDING.yml

如果有 GitHub Sponsors 或其他赞助渠道，加一行配置即可。

### 5. aweswitch README.ai.md

如果 aweswitch 也支持 AI agent 安装，参照 awescholar 添加。

## 不需要

- **CODE_OF_CONDUCT.md** — 个人项目，非社区驱动，形式大于内容。
- **GOVERNANCE.md** — 大型社区项目才需要，个人项目不适用。
